ABOUT THE APP

The AlphaRun Web Browser for Web App Developers is a simple, lightweight, full-screen browser for the Android platform. It is designed for use by developers of applications built using standard Web technologies, including HTML, CSS, and JavaScript. It is NOT a general purpose browser for normal web browsing.

While generally useful for HTML5 web app developers, AlphaRun's main purpose is for testing and running mobile applications built using Alpha Software's Alpha Anywhere development system. Alpha Anywhere is a complete system for rapidly building and deploying HTML5 business applications for mobile devices and personal computers. For more information about Alpha Anywhere, see www.alphasoftware.com.

The features include:

  • Full screen browsing
  • Only reloads URL while running when you or the page requests a reload
  • Optional use of PhoneGap/Cordova WebView with various plug-ins instead of basic WebView
  • Manually ordered bookmark list, no address bar
  • Can hide device status bar
  • "Touch Highlighting" mode displays finger touches for more effective demos on mirrored external displays
  • Can display and clear cookies
  • Can add browser load errors and current value of optional JavaScript expression to a log file
  • URL variable to insert into bookmark URLs for easier use on different LANs
  • Ability to use the camera to scan a QR code to input a URL (some development systems display QR codes on the screen of URLs you can use for testing)

SETTING, UPDATING, AND USING BOOKMARKS

The Bookmarks list shows all of the current web page bookmarks, one per row. Both the user-provided bookmark name and the URL it refers to are displayed. These bookmarks are only for this app and are not related to any other app's bookmarks.


There are two buttons above the list: Add and Preferences . The Add button opens the New Bookmark page to create a new bookmark. The Preferences button opens the AlphaRun preferences screen, where you can set browsing behaviors and access debugging logs, among other options (see below for more details).


By holding down on a specific bookmark for a short period, you are able to access functionality specific to that highlighted bookmark. Those are:

  • - Edit the highlighted bookmark's name, URL, UID, and/or ability to be updated by scanning a QR code.
  • - Delete the highlighted bookmark.
  • - Move the bookmark up the list.
  • - Move the bookmark down the list.

By pressing either the Add button (for a new bookmark) or the Edit button (for an existing bookmark) switches to the Details display to let you set the properties.

The bookmark properties that you can set in the Details view are:
Name
An optional user-provided description of the bookmark.
URL
The URL to access when the bookmark is selected. If this value does not contain "://", the characters "http://" will automatically be assumed to start the URL.
UID
An optional user-provided text value used to identify a particular bookmark. This is unused at this point.
Scan Button
If this setting is "ON", then a Scan button will appear on the right side of the row for this bookmark in the Bookmarks list. The Scan button lets you, with one tap in the Bookmarks list, update the URL for that bookmark from a QR code and then select it for display. (Some development systems display QR codes on the screen of URLs you can use for testing.)
Scan QR code URL
This is a button that displays at the bottom of the screen. Pressing it will bring up a live-view from the camera. If you point the camera at a QR code, the resulting scanned value will replace the URL value in the Details for this bookmark.

To save the current bookmark values, press the Save button .
To cancel out of the current screen without saving any changes made, press the Cancel button .

When the plain Bookmarks list is displayed, tapping on a bookmark will select it and switch to the Main Web Viewing Display, which will then load that URL.

MAIN WEB VIEWING DISPLAY AND THE TOOLBAR

The Main Web Viewing Display takes up the whole screen. It acts like a regular browser view, accepting taps, swipes, etc. It includes the Toolbar Button on the right that is used to access the AlphaRun features:

When you tap the Toolbar Button, the full Toolbar will be displayed:

Tapping the toolbar icon on the right again will hide the toolbar.

Press and holding on the toolbar button and then dragging up or down lets you reposition the toolbar button (and the toolbar when it is displayed) to different parts of the screen so as not to obscure important buttons on the web page itself. These positions are remembered separately for portrait and landscape device orientation.

The buttons on the toolbar are as follows (from left to right):

Back
Acts like a browser Back button
Forward
Acts like a browser Forward button
Refresh
Reloads the current browser display
Stop
Stops loading
Settings
Brings up the AlphaRun Settings page
Bookmarks
Brings up the AlphaRun Bookmarks list
Compass
Hides the toolbar when tapped, repositions when you press, hold, and drag

There is also an optional Javascript button that appears when the JavaScript setting is not blank. See the help for the JavaScript setting below for more information.

SETTINGS

Tapping the toolbar Settings button brings up the Settings page:



The settings are as follows:

Touch Highlighting
When ON, a small red dot will be displayed over the Main Web Viewing Display whenever your finger is on the screen. This is useful when you are mirroring the screen to an external display, such as with supported HDMI adapters.
Status Bar
Controls the display of the device status bar at the top of the screen on the Main Web Viewing Display.
Toolbar Button
Controls the display of the Toolbar Button on the Main Web Viewing Display. While you will no longer be able to access the Toolbar Button in the current Web Viewing Display, you can still access the Preferences screen from the Bookmarks list.
Use PhoneGap/Cordova
Controls which type of browser view is used. When "OFF" (the initial default), a normal, plain WebView is used. This should behave like a normal browser window. When "ON", a Cordova view is used (controlled by an Android CordovaInterface). (Cordova is the name of the Open Source engine that powers PhoneGap.) This lets the web pages you load take advantage of native Android code using Cordova (PhoneGap) protocols. You can find out which version of Cordova is being used, and see a list of the installed plug-ins, by tapping the About setting row.

Note: This setting applies to all URLs loaded by the app. When you change this setting the app will switch which view is displayed, but that view is not reloaded. You should then go to the Bookmarks list and select the URL you wish to load.
Cookies


Displays the current web cookies stored by this app. When they are displayed there is a button for deleting all of them. You may copy any of the data displayed by pressing and holding on the display and using the standard Android Copy, etc., commands.
You can view the cookies either as a selectable list per entry or one a single page with entries appropriately separated. To switch between views, you press the Switch View button . You can also delete all of the cookies by clicking on the Trash icon . These will be grayed out when the functionality is unavailable (e.g. there are no entries to display).
Log File
Brings up the AlphaRun Log File. Items are added to this file when logging errors and when the special JavaScript is run. For information about both cases, see below. When the Log File is displayed there is a button to erase all of it. You may copy any of the data displayed by pressing and holding on the display and using the standard Android Copy, etc., commands.
Navigation and display of the log entries is the same as that available for cookies.
Log Errors
Controls whether or not loading errors on the Main Web View Display are added to the Log File.
Clear Browser Cache
When tapped, this setting will remove all responses from the cache used by this app's browser views, as well as related cookies and other data. The size of the cache shown is inexact but instead a close approximation of data stored on the device from the viewing session. This cache is separate from that stored by the other brower(s) on the device.
URL Variable
This optional text is automatically inserted into URLs in place of the characters "[[v1]]". It is useful, for example, to set a part of multiple bookmarks that varies from time to time, such as the IP address of a local server on a LAN accessed over WiFi during testing.
JavaScript
This feature is mainly for developers who are familiar with JavaScript. It may be ignored by other users. When this setting has a text value, such as:
"body="+document.body.innerHTML
the Toolbar will have an extra button: the js button . Whenever that button is pressed, the JavaScript in this setting will be executed and the resulting value will be added to the Log File.
Previous JS Commands list.
A selectable list of the 5 most recent JS commands used by you. Selecting one will replace the current Javascript text field with this value.
Help
Loads and displays the latest help page from the AlphaRun support web site.
About
Displays the current app version. When tapped, it displays additional information if appropriate.

When available, the Up button next to AlphaRun's icon allows you to navigate back from your current position. The behavior should feel similar to standard Android back behavior in most circumstances.

Pressing the Android back button returns to the Main Web Viewing Display. All changes made to the Preferences will be saved.
